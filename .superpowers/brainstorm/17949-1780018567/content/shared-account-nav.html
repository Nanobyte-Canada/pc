Shared Account Navigation Bar

A reusable top bar component used by both Portfolio and Wheel Strategy pages. Shows account pills with broker icon + type + masked number. Consistent across desktop and mobile.

Concept

One component, both pages. Extract the account switcher into a shared AccountNavBar component. It renders pill tabs on desktop and a dropdown + dots on mobile. Both Portfolio and Wheel Strategy pages use the same component, ensuring identical look and behavior. Each pill shows broker icon + account type + masked ••number.

AccountNavBar Component Spec

  • File: components/layout/AccountNavBar.tsx + AccountNavBar.css
  • Props: selectedId: number | null, onSelect: (id: number | null) => void, accounts: DashboardAccount[]
  • Desktop (≥768px): Pill tabs in a container with background/border. "All Accounts" first, then per-account pills with broker icon (14px colored badge) + type + ••number (10px mono muted). Active pill: emerald bg + text.
  • Mobile (<768px): Centered dropdown trigger ("ALL ACCOUNTS" or "Q TFSA ••0190" with chevron) + dot indicators below. Tapping opens bottom sheet with account list.
  • Account number format: Always show •• + last 4 digits in monospace muted text
  • Used by: DashboardPage (Portfolio) and WheelPage — both import the same component
  • Replaces: Inline account switcher code in DashboardPage + .wheel-account-tabs in WheelPage
