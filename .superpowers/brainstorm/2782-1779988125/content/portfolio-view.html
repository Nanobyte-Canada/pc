Portfolio View — Unified Account Switching

Single screen with account switcher — view all accounts or drill into any individual account. Rename "Dashboard" tab to "Portfolio".

Concept

One screen, all contexts. The "Portfolio" screen replaces both the current Dashboard and Account Detail pages. An account switcher at the top lets users toggle between the aggregate "All Accounts" view and any individual account. The summary, KPIs, and positions update based on the selection — no page navigation, no breadcrumbs, no back button needed.

Design Decisions

  • Renamed "Dashboard" to "Portfolio" — reflects that this is a portfolio view with account switching, not a generic dashboard.
  • Account switcher (Desktop): Pill tabs in the header row, next to the title. "All Accounts" first, then individual accounts with broker icon + type. Active tab has emerald background. When individual account selected, shows masked ••number.
  • Account switcher (Mobile): Dropdown at top center (like the reference images). Shows "ALL ACCOUNTS" or "TFSA ••0190" with chevron. Dot indicators below show which account is active. Tapping the dropdown opens a bottom sheet to select a different account.
  • Context-dependent content:
    • "All Accounts": 5 KPI cards + Positions table (Holdings/Orders)
    • Individual account: 4 KPI cards (Total Value highlighted, Investment, Cash, Returns) + content tabs (Positions/Activities/Dividends)
  • No page navigation: Switching accounts is state change, not route change. No breadcrumbs, no back button. One URL: /portfolio. The selected account is stored in state (or URL query param for shareability).
  • Bottom tab bar: Portfolio | Wheel | Connections | More
